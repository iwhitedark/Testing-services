# Итоговый проект: Тестирование сервисов и приложений

## Введение

В рамках данного проекта вам предстоит написать автоматизированные тесты для веб- и мобильного приложения. Задания имитируют реальную задачу: необходимо проверить корректность работы пользовательских интерфейсов сайта и Android-приложения. Для этого используйте знания из пройденных модулей: Java, Selenium, TestNG для веб-тестирования и Appium для мобильного тестирования. Результатом будет готовый проект с тестовыми сценариями, который можно добавить в портфолио.

---

## Что нужно сделать

### Часть 1. Автотесты сайта

Выберите публичный сайт с простым пользовательским интерфейсом (например, информационный портал, онлайн-каталог или новостной сайт без авторизации и сложной бизнес-логики).

Изучите основные сценарии работы сайта (навигация по разделам, поиск, работа форм и так далее) и реализуйте для них автоматизированные тесты на Selenium.

Покройте хотя бы четыре ключевых сценария (но лучше больше). Например:

* при клике на пункт меню открывается правильная страница;
* форма поиска возвращает результаты;
* элементы на странице отображаются корректно.

---

### Часть 2. Автотесты Android-приложения Wikipedia

Напишите автоматизированные тесты для Android-приложения **Wikipedia** с помощью Appium.

Возможные задачи:

* проверка отображения главного экрана;
* ввод текста в строку поиска;
* открытие статьи и проверка её заголовка;
* переключение языковых настроек;
* прокрутка страницы.

Убедитесь, что тесты выполняются на Android-эмуляторе и стабильно проверяют заданные сценарии.

---

## Обязательные требования к покрытию

* реализуйте **не менее двух стабильных сценариев**;
* для максимального балла рекомендуется **три и более сценария**.

### Примеры сценариев

* поиск статьи по ключевому слову и проверка корректной страницы;
* проверка заголовка открытой статьи;
* прокрутка статьи до определённого элемента и проверка его наличия;
* смена языка и проверка корректной работы поиска/контента.

Тесты должны стабильно выполняться на эмуляторе Android (или реальном устройстве) и проверять ожидаемый результат.

---

## Технические требования

### Стек технологий

* язык: **Java** (рекомендуется версия 11+);
* сборка: **Maven** или **Gradle**.

### Фреймворки

* веб-тесты: Selenium WebDriver, TestNG;
* мобильные тесты: Appium Java Client, TestNG.

Рекомендуется использовать **WebDriverManager** для автоматической загрузки драйверов браузеров.

---

## Мобильное тестирование

* используйте Appium Server (локальный или Appium Desktop);
* тесты должны выполняться на Android-эмуляторе или реальном устройстве;
* предварительно установите APK Wikipedia (из официального источника).

---

## Структура и минимальные требования

* проект организован как тестовый;
* разумное разделение на пакеты (pages, tests, utils и т.д.);
* для веб-тестов рекомендуется использовать **Page Object Model**;
* архитектура может быть базовой, но код должен быть читаемым и поддерживаемым.

---

## Дополнительно (по желанию)

* можно добавить простые API-тесты (проверка ответа открытого API);
* основной фокус — UI-тестирование;
* API-тесты не обязательны, но могут положительно повлиять на оценку.

## Этапы выполнения проекта

### 1. Подготовка окружения

* установите JDK 11+;
* настройте Maven или Gradle;
* добавьте зависимости Selenium, TestNG, Appium;
* организуйте структуру проекта (например, `src/test/java`).

### 2. Исследование сайта

* выберите сайт;
* определите элементы интерфейса для тестирования;
* сформулируйте тестовые сценарии.

### 3. Разработка веб-тестов

* создайте Page Object классы;
* напишите тесты на TestNG;
* используйте ожидания и проверки (assert).

### 4. Настройка Appium

* запустите Appium Server;
* настройте Android-эмулятор или устройство;
* установите приложение Wikipedia;
* настройте capabilities (appPackage, appActivity или APK).

### 5. Разработка мобильных тестов

* откройте приложение;
* дождитесь загрузки главного экрана;
* выполните поиск статьи;
* откройте результат и проверьте заголовок.

### 6. Запуск и отладка

* запускайте тесты через TestNG XML или IDE;
* исправляйте локаторы и синхронизацию;
* добейтесь стабильного выполнения.

### 7. Документирование

В `README.md` опишите:

* зависимости;
* способы запуска тестов (`mvn test` и т.д.);
* поддерживаемые браузеры и устройства;
* примеры отчётов.

---

## Структура проекта и рекомендации

* используйте Page Object Model;
* разделяйте пакеты (tests.web / tests.mobile);
* конфигурацию храните в `config.properties`;
* применяйте аннотации TestNG (`@BeforeClass`, `@AfterClass`, `@Test`);
* используйте DataProvider при необходимости;
* работайте с явными ожиданиями (WebDriverWait);
* следите за качеством кода.
